# 新規プロジェクション(読みモデル)の追加

CQRSの読み側を規約準拠で追加する手順。既存のプロジェクション(あれば `warehouse-query/` 配下)をテンプレートとして参照する。作業前に `.claude/rules/` の cqrs-projection / testing を読むこと。

## 手順

1. ビューの用途(何を照会したいか)、必要なフィールド、対象となるイベント、クエリのパラメータをユーザーに確認する。用途ごとにビューを分ける(万能ビューにしない)。

2. **読みモデル(ビュー)エンティティ**を作成する。表示・検索に最適化した非正規化構造にし、ビジネスロジック/不変条件は持たせない。命名は用途を表す(例: `AvailableStockView` / `AllocationView` / `StockLedgerView`)。

3. **プロジェクション**を作成する:
   - 対象イベントの `@EventHandler` を実装し、読みモデルを更新する。
   - **冪等**にする(同一イベントの二重適用で壊れない。upsert / 既存チェック等)。
   - イベントストア(真実の源泉)から再構築できる状態に保つ。

4. **クエリハンドラ**を作成する:
   - `@QueryHandler` で読みモデルを返す。クエリ処理でイベントを発行しない(CQS)。

5. **テスト**を作成する:
   - 「イベント入力 → 読みモデル状態」を検証する。
   - 冪等性(同一イベント二重適用で結果が変わらない)を検証する。

6. 生成後、`./gradlew compileJava compileTestJava test` を実行して型エラー・テスト失敗がないことを確認する。

7. **コミット前に `es-domain-reviewer` を run_in_background:false で実行**し、ルール準拠(REVIEW_VERDICT: PASS)を確認する。
