# 集約設計

- 集約は整合性境界（invariant を守る最小単位）。不変条件はすべて集約内で完結させる。
- 1トランザクションで変更する集約は1つに限る。複数集約にまたがる整合はイベント＋ポリシー（Saga/Process Manager）で結果整合にする。
- 他集約は識別子（ID）で参照する。集約インスタンスを直接保持・参照しない。
- 不変条件の判断は `@CommandHandler` で行い、違反時は例外を投げてイベントを発行しない（例: `available < 要求量` なら引当を拒否）。
- 集約は小さく保つ。ライフサイクル・トランザクション頻度が異なるものは別集約に分ける。
- 集約識別子は `@AggregateIdentifier`。状態変更は必ず `AggregateLifecycle.apply(event)` を経由する（直接フィールド代入で状態を変えない）。
