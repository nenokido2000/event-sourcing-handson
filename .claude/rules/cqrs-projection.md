# CQRS / プロジェクション

- 読みモデル（Query側）は書きモデル（集約）から独立させ、専用ストア（PostgreSQL）に持つ。
- 読みモデルは使い捨て可能・再構築可能に保つ（イベント再生でいつでも作り直せる）。真実の源泉はイベントストアのみ。
- プロジェクションのイベント適用は冪等にする（同一イベントの二重適用で壊れない）。
- クエリ側にビジネスルール／不変条件を置かない。表示・検索の都合に最適化した非正規化ビューに徹する。
- ビューは用途ごとに分ける（例: `AvailableStockView`, `AllocationView`, `StockLedgerView`）。1つの万能ビューに寄せない。
- 「コマンドはイベントを生む・クエリは状態を返す」を厳守（CQS）。クエリ処理でイベントを発行しない。
