# イベントソーシング

- イベントは不変（immutable）。一度書き込んだイベントは更新・削除しない。訂正は打ち消し/補正イベントで表現する。
- イベント名は過去形・完了した事実として命名する（例: `StockReceived`, `StockAllocated`）。コマンド（意図）と混同しない。
- 集約の状態はイベントのリプレイで導出する。現在状態のスナップショットを真実の源泉として永続化しない（スナップショットは最適化目的に限る）。
- `@EventSourcingHandler` は状態遷移（フィールド更新）のみを行う。判断・バリデーション・外部呼び出し・副作用を書かない（決定は `@CommandHandler` の責務）。
- イベントは「何が起きたか」だけを持つ。UI都合・クエリ都合のデータを混ぜない。
- スキーマ進化はイベントの後方互換を守る。破壊的変更が要る場合はリビジョン＋アップキャスタで対応し、既存イベントは書き換えない。
- 1コマンドが生成するイベント列は原子的に追記する（部分適用を残さない）。
